# Отчёт по лабораторной работе №4

Проект: LR4 — ленивые последовательности и потоки (C++17)

Кратко
- Реализованы: `Sequence<T>` (абстрактный), `LazySequence<T>` (ленивая последовательность с мемоизацией), `MaterializedSequence<T>` (полностью материализованная последовательность) и alias `EagerSequence<T>` как более читаемое имя для материализованной последовательности.
- Реализовано семейство потоков: `ReadOnlyStream<T>` и `WriteOnlyStream<T>`.
- Реализована задача: синхронизация временных потоков по опорному потоку с окном `tau`.
- Написаны модульные тесты (Catch2) и интегрирован единый исполняемый файл `lr4`, в который встроен тестовый раннер. Запуск тестов: `lr4 --run-tests`.
- Добавлены удобные сборочные скрипты `build.ps1`/`build.sh` и верхнеуровневый `CMakeLists.txt`.
- Добавлен CI workflow (см. `.github/workflows/ci.yml`).

Что сделано (подробно)
- `cpp/include/sequence.hpp` — `Sequence<T>` (интерфейс), `MaterializedSequence<T>` (внутренняя векторная реализация), алиас `EagerSequence<T>`.
- `cpp/include/lazy.hpp` — `LazySequence<T>` с генератором `std::function<T(size_t)>`, операциями `Map`, `Zip`, `Append`, `GetSubsequence`.
- `cpp/include/types.hpp` — `Event<T>`: пара `(double t, T value)`.
- `cpp/include/stream.hpp` — `ReadOnlyStream<T>`, `WriteOnlyStream<T>` обёртки вокруг `Sequence<Event<T>>`.
- `cpp/include/sync.hpp` — `SynchronizeStreams<T>`: простая, корректная реализация синхронизации по референсному потоку и окну `tau`.
- `cpp/src/main.cpp` + `cpp/src/cli.cpp` — CLI: интерактивный режим (ручное тестирование) и демонстрация.
- `cpp/tests/` — `test_lazy.cpp`, `test_streams.cpp` — набор тестов Catch2, покрывающих основные операции и сценарий синхронизации.

```powershell

```

Пояснение термина "окно" (`tau`)
- Окно `tau` — максимальная допустимая задержка между моментом события в референсном потоке и считающимся соответствующим событием во вспомогательном потоке. Для каждого события `t_ref` из референсного потока выбирается первое событие `t` во вспомогательном потоке такое, что `t_ref <= t <= t_ref + tau`.

Совместимость с требованиями ТЗ — чек-лист

1) Реализация `LazySequence<T>` — выполнена: `cpp/include/lazy.hpp`. Поддерживает генератор, мемоизацию, операции Map/Zip/Append/GetSubsequence.
2) Реализация семейства `Stream<T>` — выполнена: `cpp/include/stream.hpp` (ReadOnly/WriteOnly).
3) Тематическая задача — выполнена: синхронизация временных потоков (`cpp/include/sync.hpp`). Результат материализуется в последовательность кортежей `(t_ref, vector<Option<T>>)`.
4) Модульные тесты — выполнены: `cpp/tests/*` + встроенный раннер; покрывают основные сценарии и границы в тестах.
5) CLI (ручное тестирование) — реализовано: интерактивный режим в `lr4.exe`.
6) Автоматическое тестирование — реализовано через `--run-tests`; CI workflow добавлен.
7) Тестирование на предельных значениях — частично: структура позволяет генерировать большие последовательности, но в рамках текущего POC добавлены функциональные тесты; при желании добавлю стресс‑тесты для миллионов элементов.
