# Отчёт по лабораторной работе №4

## Цель работы
Реализовать `LazySequence<T>`, семейство `Stream<T>` и тематическую задачу на синхронизацию временных потоков. Проект должен поддерживать ручной режим ввода, работу с файлами, модульные тесты и автоматический запуск проверок.

## Выбранный вариант
Вариант 5.1 — синхронизация потоков временных данных.

## Что реализовано
- `Sequence<T>` — базовый интерфейс последовательности.
- `VectorSequence<T>` — конкретная реализация на основе `std::vector`.
- `LazySequence<T>` — ленивая последовательность с мемоизацией уже вычисленных элементов.
- `Stream<T>` — абстрактная база для потоков.
- `ReadOnlyStream<T>` — поток чтения из последовательности или файла.
- `WriteOnlyStream<T>` — поток записи в память или файл.
- `SynchronizeStreams<T>` — алгоритм синхронизации временных потоков по опорному потоку и окну `tau`.
- CLI — интерактивный режим для ручной проверки.
- Тесты — Catch2, запуск встроен в `lr4.exe` через `--run-tests`.
- Makefile — единая точка входа `make` для сборки, запуска и тестирования.

## Структура проекта
- `cpp/include/sequence.hpp` — только интерфейс `Sequence<T>`.
- `cpp/include/vector_sequence.hpp` — реализация `VectorSequence<T>`.
- `cpp/include/lazy.hpp` — реализация `LazySequence<T>`.
- `cpp/include/stream.hpp` — базовый абстрактный класс `Stream<T>`.
- `cpp/include/read_only_stream.hpp` и `cpp/src/read_only_stream.cpp` — поток чтения.
- `cpp/include/write_only_stream.hpp` и `cpp/src/write_only_stream.cpp` — поток записи.
- `cpp/include/sync.hpp` — синхронизация потоков.
- `cpp/src/main.cpp` — основной цикл CLI.
- `cpp/src/cli.cpp` — меню и разбор пользовательского ввода.
- `cpp/src/embedded_tests.cpp` — запуск Catch2-тестов.
- `cpp/tests/test_lazy.cpp` и `cpp/tests/test_streams.cpp` — модульные тесты.
- `Makefile` — короткие команды `make`, `make test`, `make run`.

## Сборка и запуск
Сборка выполняется через `make`:

```powershell
cd lr4
make
```

Проверка и запуск:

```powershell
cd lr4
make test
make run
```

Если нужен прямой вызов через CMake, это тоже поддерживается, но основной путь теперь — `make`.

## Как работает `tau`
`tau` — это окно синхронизации. Для опорного события с временем `t_ref` во вспомогательном потоке ищется первое событие `t`, для которого выполняется `t_ref <= t <= t_ref + tau`. Если событие найдено, его значение входит в результат, иначе ставится `nullopt`.

## Работа с файлами
- Поток чтения может быть создан из файла и парсить строки в `Event<T>`.
- Поток записи может писать события в файл и параллельно хранить буфер в памяти.
- В CLI добавлен file-demo, а в тестах есть проверка roundtrip для файлового режима.
